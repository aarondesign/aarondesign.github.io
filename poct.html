Point of Care Test Project

Point of care test project includes a portable device and web platforms. The object of this product is to measure the patients’ lipid and Alc level in the blood. In this project, my job is to design the app and web system including flows and user interface.

Not only design the product, but I also go to the hospital to observe and interview with some patients. Then report some usability issues to the managers and give some suggestions about how to improve the product.

User's Journey

Before designing the UI flows, the first thing is to figure out the scenario that this product will be used by what kind of people and what kind of benefit we can provide.

In this project, the healthcare center will buy this product to measure the customs’ blood. According to the test result, the system will give custom health advice, health supplements, and test records.

After some months, the system will remind customs to come back for follow up at the healthcare center.

Design Concept

Before designing the user interface on the device's screen, I will consider who would use the product to make jobs done during particular circumstances. On this project pharmacists will use the device in different ways, I should consider the situations when designing the user interface.

I place the main control element on the bottom of the device so that the pharmacists could tap the buttons quickly. About the color scheme, I select a high contrast color for readability. Prevent using red(danger)/yellow(alert)/green(normal) as main color, therefore I pick blue color.

Design Review

When designing the user interface of the medical device, the most important thing is not what it look but the regulation. The first version ui show the different color according to the different level of result. After an interview with stakeholders, they suggesgt the result page show the number only because the device should not diagnose the result but the doctors or pharmacists do it.
