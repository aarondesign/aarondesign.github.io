App Flow

Improved the flow of sign-up and on-boarding steps

The problems of previous flow

Previous on-boarding flow was very complicated, including sign-up, filling cycle info, and personal info. Especially, the App should validate the account by Email. Users must check the validation code in their inbox and then go back to the App. It took some time to finished these tasks, and some users would give up and leave our App.




Improved - Info user this app should has to go with test strip

In the past version, App didn't tell users that it should be used width test strips. Therefore, the latest version shows that the App should be used with strips and tell users the shop info according to their sim card information.

Improved - To minimize the on-boarding barriers

Just query two essential questions:＃1 Average Cycle days and #2 The first day of last period. According to the information, the App will suggest users when to measure. I planned the sign-up process after users using this App, and the trigger is that users should sign up then they could check the measure records.


Welcome on board

Set welcome page as a breakpoint before entering the home screen. Let users understand that they completed the on-boarding process and info users when the next measure time comes.

Sign up trigger

Place the sign-up process after users experienced the App, and then request users to register by some trigger - Check the measure records.

Pin a card on the top of the home screen that reminds users to sign-up.

Sign up to view the records。

Improved - Permissions Request


Problems on previous version

App will ask for permissions when user launched the app in the beginning, including camera, notification and photos. Users couldn't figure out why they should commit the request without any context.


Improved - Ask for permissions in context and communicate the value the access will provide.

  2. Camera：App will ask for the camera permission at first time of measurement and app will explain the purpose. 。

Measurement Flow



Measurement is the key experience of this app.
Improve the flow and interface will enhance the users' experience.

Remind to measure

There is a card belong to measurement on the home screen. The card shows the date of the next measure day, and it will remind the user to measure when the day comes.




Figure out the position of front camera ( Android )

The App must utilize the front camera to analyze the strip. But there are kinds of Android Phones, and we couldn't indicate the front camera position by engineering method. Therefore, I designed a flow that request users to point out the camera position before measurement.

    Steps:
  1. After onboarding, there will show a dialog that asks the user to set the camera.
  2. Announce that App doesn't support the pop-up camera.
  3. Place the finger over the front camera and wait.
  4. Check the position of finger.
  5. Swipe the finger from the camera to the screen area.
  6. Drag the pin slightly to aim the camera.
  7. Inform user that camera has been set.




Place the strip ( Android )

Place the strip on the camera is the key step to measure the test strip.


  1. Guide user place strip to the specific area step-by-step
  2. We found that users would push the strip to the rudely; it would result in mistakes of measurement. We designed a method that guides users to place the strip in two separated steps.
  3. Except for visual and sound feedback, I added Haptic feedback when users placed the strip to the right place.
Show the test result on the home screen.

❌ Previous version: Closing the result page and go back to the home screen, it wouldn't show the result on the home screen. Users should check the result on the records page.

✅ Redesigned: Show the result on the home screen; therefore, users could read the measurement result intuitively

Home Screen

The home screen provides essential information, and users will interact with the app here frequently. The redesigned home screen offers users information about the current body status, and it also guides users to complete the missions.

  1. ① Previous design shows the date of today. ➡️ Redesign: Tap the card link to the body log page that we hope users could interact with the app every single day.

  2. ② The circle has two sides. Tap it would flip 180 degrees and shows the other side, one side shows the days of next fertile windows, another side shows the start date of the next period. ➡️ Resigned: Separate the info into two different cards. The first one displays the next period info, and the second one shows the following fertile windows info.

  3. ③ Place the measure button to Bottom Tab.

  4. Using the buttons link to other pages in the previous design. ➡️ Change to bottom Tab, it's convenient for users to switch in different page, Home Screen, ④ Records, ⑤ Calendar and ⑦ Settings.

  5. I removed the past notification message page ⑥. The App integrated the notification message with the ② card, and it doesn't make sense that shows the prior notifications on this app. Thus, I remove it.

Inspiration from project


1️⃣Don't forget some unobvious features
Designers should consider some obscure features when planning an app, including (1) suggest users to update the app. (2)Invite users to review the App after they experienced the App's better service. (3)A method to info users that the App will update terms and policy. (4) Design key metrics to evaluate the App, we could improve the App according to the metrics. (5) To optimize the app search, then users could find the App by suitable keywords.


2️⃣ Lower entry barrier
We want to acquire more users as possible, and hope they could interact with the app with fewer barriers. Therefore, the onboarding experience is a critical touchpoint. The redesigned App improved the onboarding experience; the App asks essential questions and postpones the sign-up process until users are familiar with our app.
