女性黃體素檢測 App 再設計

舊版的App上線使用經過了兩年，根據這段期間內收集一些使用者數據、
使用者訪談以及內外部的利害關係人回饋資訊，將 UI 的流程及整理改善使用經驗。

改版主要的目標為：
  • ✔︎ 首次使用與註冊流程的改善 - 降低使用者的進入門檻
  • ✔︎ 重新建立 UI system - 確保使用上的ㄧ致性
  • ✔︎ 調整 Layout - 重新編排資訊的呈現
  • ✔︎ 增加輔助功能 - 幫助使用者了解產品

伴隨改版完成的事項：
  • ✔︎ 產品介紹頁面的重新設計
  • ✔︎ App 驗證碼通知郵件的版面設計

App 流程

首次使用與註冊使用流程的改善

原流程問題

上一個版本的首次使用 ( Onboarding ) 流程相當的繁瑣，包含 (1) 註冊、填寫 (2) 經期資訊、(3) 個人基本資料。步驟之多，尤其當填寫完帳號之後需要離開App，打開信箱App查閱驗證碼，接著再回到App。 這段過程需要花費許多時間，有些沒有耐性的使用者應該已經悻悻然的關掉App。




改善 - 預先告知 App 需要搭配試片使用

在首次使用告知使用者，需要先購買試片。並根據Sim卡內的資訊判斷使用者所在的地區，提供相對應的購買資訊連結。

改善 - 降低首次使用門檻

首次使用只詢問兩項必要資訊：
( 1 ) 週期天數以及( 2 ) 上次經期的初始日，這兩個資訊為系統計算受孕期的必要條件。
而註冊流程以及基本資料的填寫流程則置於使用者使用系統後，根據脈絡引導使用者註及填寫基本資料。


首次使用畫面

首次使用的畫面，只需要讓使用者填寫必要資訊。在正式進入到系統之前，規劃了一個歡迎頁面最為一個停頓點，讓使用者知道流程完成，順便告知使者下一次預訂量測的時間。

註冊 - 需要再註冊即可

將註冊流程置於首次使用之後: (1) 在首頁的置頂處，放置一張卡片，提醒使用者註冊。 (2)在量測紀錄頁面。如果使用者想要觀看紀錄，需要先註冊才能檢視。


延後註冊的目的，希望使用者先行體驗整個 App的流程與服務，在合理的脈絡下要求使用者註冊。

註冊才可查看歷史紀錄。

改善請求手機權限的流程


原流程問題

原流程會在使用者開啟手機的第一個畫面，就向使用者要求所有必要的權限，包含 (1) 相機的權限、 (2) 通知權限、(3) 相簿的存取權限。使用者在還不了解App所有功能時，一次性的的要求數個權限，會讓使用者摸不著頭緒。


改善 - 在脈絡下要求權限

  1. 通知權限：在首次使用的歡迎頁，會出現請求通知的對話。
  2. 相機權限：在第一次量測前，會出現請求相機權限的對話。
  3. 相簿存取權限：在第一次使用分享功能時，會出現權限的請求對話。

改善量測流程



量測為App的最核心體驗項目，改善量測的流程與量測結果的顯示，將有助於提升使用經驗，減少客訴。

改善 - 提醒量測

在App的首頁設計一張屬於量測的卡片，讓使用者可以更明確的知道，距離量測日還有幾天以及量測日期。當到了該進行量測的當日，卡片會提醒使用者今天為量測日。

改善 - 設定前鏡頭位置 ( Android )

試片需要透過手機的前鏡頭進行分析，但是Android 的機型眾多，其前鏡頭位置各有不同。無法透過程式一一設定。因此我們設計了一套方法邀請使用者，在量測前確認前鏡頭的位置。



  1. 在首次進到首頁，會請求使用者設定相機。
  2. 宣告設定的目的，以及說明不支援的機型
  3. 請求使用者將手指至於前鏡頭上。
    (上一個版本的說明圖，手指畫得跟真實比例太接近，在測試時發現有使用者將手指置於圖上。)
  4. 請使用者將手指從前鏡頭的位置，向下滑入螢幕。
  5. 左右移動拉桿，確定對準前鏡頭。
  6. 確認畫面，跟使用傳達相機的設定已經完成，達到良好的溝通。

App 流程


改善 - 分段指引使用者放置試片 ( Android )

放置試片是App功能中的關鍵體驗，但涉及到與硬體的互動，所以行為上又更為複雜些。



  1. 一步一步的引導使用者準備試片，放置到指定位置。
  2. 在使用者測試時，發現使用者會將試片直接推到底，造成試片量測失敗。
    這次改版，請使用者兩段式的將試片放置到指定位置，減少試片推到底的機會。
  3. 增加觸覺 (Haptic )回饋。當使用者將試片推至正確位置，除了視覺上的回饋外，也加上了觸覺與聽覺的回饋。
改善 - 將當日結果顯示於首頁

❌ 在原先的流程上，當量測完畢，回到首頁後就像啥事都沒發生一樣，使用者需要到量測紀錄找尋今日的結果。

✅ 流程修正後，當量測完成，回到首頁會顯示今日的結果，並顯示相關的說明文章，幫助使用者更了解量測所代表的意義為何。

Layout Redesigned

首頁

首頁是乘載最重要資訊，以及與使用者互動最為頻繁的頁面。重新調整版面目的在，提供使用者足夠的脈絡完成任務。

  1. ① 日期指顯示當日年月日，改版後變為卡片形式，點擊可以連結到當日健康紀錄

  2. ② 指示使用者距離經期開始/受孕期還有多久，從原本單一圓圈的形式改變為兩張卡片。一張為距離下次經期開始，另一張為距離下一次受孕期，距離較近的卡片放置在上方。

  3. ③ 將量測的按鈕與放置到 Bottom Tab 的位階。

  4. 將主要的頁面切換，由原本的存粹按鈕的形式，變更為Bottom Tab，方便使用者在主要的頁面中切換：首頁、④紀錄、⑤日曆、⑦設定

  5. 取消歷史訊息的頁面⑥。App的訊息通知為提醒量測、提醒記錄經期，這兩項通知整合到②的卡片上，而且歷史的通知訊息對於使用者來說並無太多意義，因此將此功能取消。
量測紀錄頁面
  • ❌ 上一個版本的量測紀錄頁面的排序，僅為依照時間的排列，由新到舊。

  • ✅ 新版本的量測頁面，將同一週期內所量測的紀錄放置在同一張卡片內，並且標示週期的初始日期。 如此整理，方便使用者閱讀以及比較過往的量測紀錄。

專案中獲的啟發


1️⃣別忘了 UI 以外的 feature
在設計App時會埋首於產品主要面向的流程與UI設計，但有些不常用但必要的功能需要同時被考慮進來。包括：(1)新版的的更新提醒、(2)邀約評價，需要安排使用者最滿意的當下邀請使用者給產品開發者一些鼓勵、(3)使用條款、隱私政策的變更流程、 (4)設計關鍵指標，衡量產品的成功與否、(5)設計 App 商店搜尋最佳化(ASO)，要確保使用者能透過熱門且適合的關鍵字找到 App，提高被下載的能見度。


2️⃣ 降低進入門檻
養、套、殺。在數位產產品的設計上，會希望儘可能的大量獲取夠多的使用者，讓他們無痛的開始使用產品，並且持續地與產品互動，未使用者帶來價值為服務提供者創造利潤。 因此，在產品的首次使用體驗就是一個勝負的關鍵。在這次的 App 改版於首次登入使用的流程做很很大的改變，只詢問必要的資訊，且將註冊流程挪到進入 App 之後。目的在於 讓使用者熟悉系統，等到使用者量測完成要查詢歷史資訊時，才需要註冊。
