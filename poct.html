Point of Care Test Project

Point of care test project includes a portable device and web platforms. The object of this product is to measure the patients’ lipid and Alc level in the blood. In this project, my job is to design the app and web system including flows and user interface. Not only design the product, but I also go to the hospital to observe and interview with some patients. Then report some usability issues to the managers and give some suggestions about how to improve the product.

What I do
  • 1. Mobile Device UI/UX
  • 2. Web Platform UI/UX

I plan to connect the nodes on the process systematically, but what I am response is only designing the user interface.

User's Journey

Before designing the UI flows, the first thing is to figure out the scenario that this product will be used by what kind of people and what kind of benefit we can provide.

In this project, the healthcare center will buy this product to measure the customs’ blood. According to the test result, the system will give custom health advice, health supplements, and test records.

After some months, the system will remind customs to come back for follow up at the healthcare center.

Design Concept

Before designing the user interface on the device's screen, I will consider who would use the product to make jobs done during particular circumstances. On this project, pharmacists will use the device in different ways. I should think about the situations when designing the user interface.

I place the main control element on the bottom of the device so that the pharmacists could tap the buttons quickly. About the color scheme, I select a high contrast color for readability. Prevent using red(danger)/yellow(alert)/green(normal) as main color, therefore I pick blue color.

Usability Test

During the developing phase, I join an experiment to test users in a hospital. We assign users a mission that examines the blood by themselves — observing the users while they operated the processes and filled a SUS ( System Usability Scale ) survey. I also interviewed some of them to understand the opinions of the users. I analyzed the SUS surveys and organized the users' opinions. Users will know how to operate this product, but it's hard to use by one user, they need the professional person to help them.

Design Review

When designing the user interface of the medical device, the most important thing is not what it look but the regulation. The first version UI show the different color according to the different level of result. After an interview with stakeholders, they suggest the result page show the number only because the device should not diagnose the result, but the doctors or pharmacists do it.

Web Platform

After testing, the analyzed data will sync to the web platform immediately. The pharmacies could use this web to read the data and present to the customer their health situation. Finally, print the test results and health advice for customs.

What I learn from this project

1⃣ Domain Knowledge:
What is Hab1c / Lipid / POCT stand for? When I join this project in the beginning, I am so confused with the jargons. Google these words, reading some relevant articles, searching related products, and ask my colleagues. Having basic knowledge helps me to design the product with confidence.


2️⃣ Who I design for:
There are kinds of users in this project, including pharmacists, nurses, consumers, salespersons, distributors. My role is to design the interface for them, but I rarely have the chance to interviews with them. Luckily, I have an opportunity to participate in a program to observe users using this product and discussion with participants, and I got some feedback and insights from them. In the end, I report to the stakeholders about what we should improve our product.


3️⃣ Engineers is my friends:
They would remind you that the flow or UI design will confront some technical issues, discuss with them frequently before delivering the design . In this project, there is a signal issue when the camera read the test strip. Cause the bright color background of UI to affect the signals, I switch the background to the dark color and the issue was resolved.
